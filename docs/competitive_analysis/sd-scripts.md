### 表 C：结论（优缺点与可借鉴点）

| 项目 | 结论 |
| --- | --- |
| 架构优点 | 1) 分层清晰：library 作为共享训练内核，networks 作为网络实现层，脚本层作为编排入口。 [[runcomfy.com]](https://www.runcomfy.com/comfyui-nodes/sd-lora-trainer), [[deepwiki.com]](https://deepwiki.com/edenartlab/sd-lora-trainer) 2) LoRA 训练具有“动态网络模块加载”机制，能以 network_module 方式切换实现，具备一定插件味道。 [[comfy.icu]](https://comfy.icu/extension/edenartlab__sd-lora-trainer), [[replicate.com]](https://replicate.com/edenartlab) 3) TOML 数据集蓝图机制把复杂 dataset 表达从 CLI 中抽离，利于复用与扩展。 [[pelayoarbues.com]](https://www.pelayoarbues.com/notes/Training-a-Personal-LoRA-on-Replicate-Using-FLUX.1-dev), [[github.com]](https://github.com/Akegarasu/lora-scripts) 4) SDXL 支持体系化：专用工具层封装 dual encoder 与 size conditioning，使 SDXL 训练方法族结构完整。 [[replicate.com]](https://replicate.com/cloneofsimo/lora-training) |
| 架构不足 | 1) 多脚本入口 + 海量参数：参数参考页明确规模巨大，带来学习与维护成本。 [[deepwiki.com]](https://deepwiki.com/Akegarasu/lora-scripts), [[runcomfy.com]](https://www.runcomfy.com/comfyui-nodes/sd-lora-trainer) 2) 虽有 network_module 动态加载，但整体仍以脚本编排为主，对外缺少统一的“库级 Trainer API”。 [[comfy.icu]](https://comfy.icu/extension/edenartlab__sd-lora-trainer), [[runcomfy.com]](https://www.runcomfy.com/comfyui-nodes/sd-lora-trainer) 3) 推理侧采用自定义 PipelineLike 扩展 diffusers，能力强但对外集成需要理解其自定义接口。 [[docs.eden.art]](https://docs.eden.art/tools/lora_trainer), [[onegen.ai]](https://www.onegen.ai/project/leverage-low-rank-adaptation-for-enhanced-stable-diffusion-with-sd-scripts/) |
| 可借鉴点（用于你未来的训练器库） | 1) Shared Kernel：把数据、训练循环、缓存、优化抽成稳定内核。 [[github.com]](https://github.com/Akegarasu/lora-scripts), [[onegen.ai]](https://www.onegen.ai/project/leverage-low-rank-adaptation-for-enhanced-stable-diffusion-with-sd-scripts/) 2) TOML→蓝图→对象链：用工厂/蓝图生成 DatasetGroup，既可读又适合 GUI/SDK 自动化。 [[pelayoarbues.com]](https://www.pelayoarbues.com/notes/Training-a-Personal-LoRA-on-Replicate-Using-FLUX.1-dev), [[github.com]](https://github.com/Akegarasu/lora-scripts) 3) 网络模块动态加载：将“训练编排”与“适配器实现”解耦，便于扩展 LoRA 变体。 [[comfy.icu]](https://comfy.icu/extension/edenartlab__sd-lora-trainer), [[replicate.com]](https://replicate.com/edenartlab) |
| 应避免的反模式 | “脚本越来越多 + 参数越来越多”导致复杂度外溢，参数参考页本身就反映了这种规模风险。 [[deepwiki.com]](https://deepwiki.com/Akegarasu/lora-scripts) |
