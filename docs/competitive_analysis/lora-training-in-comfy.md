### 表 C：结论（优点、不足、可借鉴点、反模式）

| 分类 | 内容 |
| --- | --- |
| 架构优点 | 1) 产品形态非常清晰：把 LoRA 训练搬进 ComfyUI，且默认保存到 loras 目录，训练后刷新即可测试，闭环体验强。 [[github.com]](https://github.com/LarryJane491/Lora-Training-in-Comfy), [[runcomfy.com]](https://www.runcomfy.com/comfyui-nodes/Lora-Training-in-Comfy) 2) 节点化增强监控：新增 Tensorboard 访问节点，降低训练过程观测成本。 [[github.com]](https://github.com/LarryJane491/Lora-Training-in-Comfy), [[comfy.icu]](https://comfy.icu/node/Lora-Training-in-ComfyUI) 3) 提供 Advanced node 扩展空间：在保持易用的同时为更进阶用户开放更多参数入口。 [[github.com]](https://github.com/LarryJane491/Lora-Training-in-Comfy), [[comfy.icu]](https://comfy.icu/node/Lora-Training-in-Comfy-Advanced) |
| 架构不足 | 1) 依赖冲突风险显著：作者明确提示可能与其他自定义节点冲突，说明依赖边界与隔离不足。 [[github.com]](https://github.com/LarryJane491/Lora-Training-in-Comfy), [[runcomfy.com]](https://www.runcomfy.com/comfyui-nodes/Lora-Training-in-Comfy) 2) 数据组织约定偏硬：强依赖“数字_名称”命名与父目录路径规则，提升易用性同时也引入约束与踩坑点。 [[github.com]](https://github.com/LarryJane491/Lora-Training-in-Comfy), [[runcomfy.com]](https://www.runcomfy.com/comfyui-nodes/Lora-Training-in-Comfy) 3) SDXL 支持不确定：作者表示无法测试 SDXL，这会影响“面向新模型族”的产品可信度。 [[github.com]](https://github.com/LarryJane491/Lora-Training-in-Comfy), [[runcomfy.com]](https://www.runcomfy.com/comfyui-nodes/Lora-Training-in-Comfy) |
| 可借鉴点（给你做训练器库） | 1) UI 闭环设计：训练产物落盘到推理工具默认目录，训练后即刻可用，是“训练器产品化”的关键体验点。 [[github.com]](https://github.com/LarryJane491/Lora-Training-in-Comfy), [[runcomfy.com]](https://www.runcomfy.com/comfyui-nodes/Lora-Training-in-Comfy) 2) 监控入口一等公民：把 Tensorboard/日志访问做成 UI/节点能力，可显著降低用户调参成本。 [[github.com]](https://github.com/LarryJane491/Lora-Training-in-Comfy), [[comfy.icu]](https://comfy.icu/node/Lora-Training-in-ComfyUI) 3) 低门槛 preset + 高级入口并存：基础节点给出少量参数，高级节点/高级配置提供更全控制，是优秀的参数分层策略。 [[github.com]](https://github.com/LarryJane491/Lora-Training-in-Comfy), [[comfy.icu]](https://comfy.icu/node/Lora-Training-in-Comfy-Advanced) |
| 应避免的反模式 | 1) 依赖无隔离导致插件冲突：如果你未来做 Python 库或可嵌入训练器，建议通过可选依赖、extras、或插件隔离机制降低冲突概率。 [[github.com]](https://github.com/LarryJane491/Lora-Training-in-Comfy), [[runcomfy.com]](https://www.runcomfy.com/comfyui-nodes/Lora-Training-in-Comfy) 2) 把高级能力留给“改脚本”：作者明确很多参数需要到 train.py 手动改，这在产品化/规模化支持时会带来维护与用户支持成本。 [[github.com]](https://github.com/LarryJane491/Lora-Training-in-Comfy), [[comfy.icu]](https://comfy.icu/node/Lora-Training-in-Comfy-Advanced) |
