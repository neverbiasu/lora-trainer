### 表 C：结论（优点、不足、可借鉴点、反模式）

| 分类 | 内容 |
| --- | --- |
| 架构优点 | 1) 容器化交付明确：Cog 打包让训练流程可在 Replicate 或本地一致运行，降低环境差异与部署成本 [[github.com]](https://github.com/replicate/lora-training/blob/main/README.md), [[github.com]](https://github.com/replicate/lora-training)。2) 入口清晰：基础/高级 predictor + common 抽取，属于典型“产品接口分层 + 内部复用”结构 [[github.com]](https://github.com/replicate/lora-training), [[github.com]](https://github.com/replicate/lora-training/blob/main/README.md)。3) 预设面向常见训练类别（face/object/style），降低非专家用户的超参决策成本，适合 API 化训练服务 [[github.com]](https://github.com/replicate/lora-training/blob/main/README.md), [[replicate.com]](https://replicate.com/cloneofsimo/lora-training)。 |
| 架构不足 | 1) “服务封装”而非“训练框架”：高度依赖固定输入 schema 与容器流程，对研究型扩展（新损失、新数据管线、新模型族）不友好 [[github.com]](https://github.com/replicate/lora-training/blob/main/README.md), [[github.com]](https://github.com/replicate/lora-training)。2) 代码规模小且以 predictor 为中心，更多体现工程交付，而不是训练系统抽象（缺少可插拔 Trainer/Dataset/Adapter 分层） [[github.com]](https://github.com/replicate/lora-training), [[github.com]](https://github.com/replicate/lora-training/blob/main/README.md)。 |
| 可借鉴点（给你做训练器库） | 1) “训练即服务”的交付思路：把训练 pipeline 封装成稳定的 API/容器接口，有助于未来做云训练或企业内训练平台 [[github.com]](https://github.com/replicate/lora-training/blob/main/README.md), [[github.com]](https://github.com/replicate/lora-training)。2) 基础/高级两层接口设计：对外提供低门槛 preset，同时保留高级参数入口，适合产品化训练器的 UX 分层 [[github.com]](https://github.com/replicate/lora-training/blob/main/README.md), [[github.com]](https://github.com/replicate/lora-training)。3) 训练与推理拆成两个 Cog 模型（training/inference）形成闭环，利于部署与权限隔离 [[github.com]](https://github.com/replicate/lora-training/blob/main/README.md), [[github.com]](https://github.com/replicate/lora-inference)。 |
| 应避免的反模式 | 1) 过度硬编码 preset 导致“通用性不足”：当你的目标是通用图像/视频 LoRA 训练库时，建议把 preset 作为可配置层，而不是把训练核心写死在 predictor 里 [[github.com]](https://github.com/replicate/lora-training/blob/main/README.md), [[github.com]](https://github.com/replicate/lora-training)。2) 只做服务封装而缺乏内部抽象：如果要做可扩展库，内部至少需要可替换的数据管线、adapter 注入与训练循环层，否则后续扩展成本会快速上升 [[github.com]](https://github.com/replicate/lora-training), [[github.com]](https://github.com/replicate/lora-training/blob/main/README.md)。 |
